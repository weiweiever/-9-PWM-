暑假机器人（避障车）：
一、总括：
	3路电机PWM输出：	
				1.PA7  TIM14 CH1	前轮
				2.PA6  TIM13 CH1	右侧车轮
				3.PB14 TIM12 CH1	左轮
	3路编码器读取：	
				1.PB6 PB7 TIM4			3900/r
				2.PC6 PC7 TIM3
				3.PA0 PA1 TIM5
	2路超声波捕获
				1.PE5 TIM9  CH1		左侧超声波
				2.PB8 TIM10 CH1		右侧超声波
	
	独立IO：
		超声波触发Trig：
				PE2 Trig1
				PE3 Trig2
		电机方向控制：
				1. PE8 PE9		前轮
				2. PE10 PE11	右轮
				3. PE12 PE13	左轮
				
	
	辅助：串口：USART1 PA9 PA10
	
	注意：3s电池可能超过12V（电机要求6V-12V），为了安全我充电都保持在+12V以内，毕竟耗电量不大
	
二、线路连接：
	PE2 PE3与两个超声波模块Trig引脚连接（任意）
	左侧超声波模块Echo连接PE5，右侧Echo连接PB8
	两个L298N的+12V接口都直接接电池正。
	左侧L298N左侧接口连接左侧电机，右侧接口连接前轮电机，	IN1-PE12
															IN2-PE13
															IN3-PE8
															IN4-PE9
															ENA-PB14  (PWM)
															ENB-PA7	  (PWM)
	右侧L298N右侧接口连接右侧电机
									IN3-PE10
									IN4-PE11
									ENB-PA6		(PWM)
	
	编码器未用到只是写了代码连了线，只单独测试过未在成熟代码上测试。
	左侧电机编码器：PA0 PA1
	前轮电机编码器：PB6 PB7	（板子上PB5也写成了PB6，注意区分）
	右侧电机编码器：PC6 PC7
	
	电源、单片机、L298N、超声波共地。
	连好电源之后右侧L298N的开关决定了单片机的供电问题，可作为小车开关。运行时记得把两个L298N的开关打开
	单片机的接地我连接的是烧程序的SW接口的GND（线长不够）
	另外注意杜邦线不稳定，，很多地方不牢固可能断开
	
三、算法描述
	LED未用到。
	motorcontrol.c中定义了	三个PID结构体
							TIM12、13、14的PWM输出Init函数，
							TIM3、4、5的编码器获取接口Init函数，
							电机方向控制IO引脚的函数（注意开漏输出）
							PID三个参数的初始化函数（未用）
							控制单个电机速度函数（未用）
							整车停止函数
							整车按某个方向运动的函数（速度如果要变的话参数可能需要调整很多次）
								方向切换加一段时间的停止
	Ultrasound.c中定义了	TIM9、10的输入捕获Init函数以及它们的中断服务函数
								（长度计算已经加入中断处理中，当前的距离存储在dis1（左）和dis2（右）变量中，主函数只需要判断这两个变量判断距离
							Trig IO引脚初始化函数
							
	PID.h中定义了			PID结构体
	PID.c中定义了			对某个PID结构体的初始化函数
							重新设置某个PID结构体PID调节目标函数（清空之前的数据）
							对某个PID结构体根据PID计算新的输出值函数（并更新数据）
							设置某个PID结构体的PID参数函数
							
	Timer.c中定义了			TIM7的5ms中断初始化函数及中断服务程序
								（每5ms读取编码器的值获取走过的长度，并对超声波模块发出一次发射命令）
								（设计了根据PID的值调节PWM值的函数，已注释）
								
	main.c中：				各种初始化
							Trig引脚置0
							针对不同的距离（dis1、dis2）设置运动方式
							为留出足够空余，一次就保证移动一定距离，也另外加了再次确认的函数，同时要考虑误差造成数据抖动。
